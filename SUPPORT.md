## Getting Help

### For Users

If you're using the Enterprise Support app and need help:

1. **Check the In-App Help:** The app includes built-in support documents and IT help resources
2. **Contact Your IT Department:** As an enterprise application, your organization's IT team is your primary support contact
3. **Review Documentation:** Check the app's documentation within the application

### For Administrators

If you're deploying or managing the Enterprise Support app:

1. **Configuration Guide:** See [docs/CONFIGURATION.md](docs/CONFIGURATION.md) for setup instructions
2. **Deployment Guide:** See [docs/iOS_DEVELOPMENT.md](docs/iOS_DEVELOPMENT.md) for deployment information
3. **Troubleshooting:** See [docs/TROUBLESHOOTING.md](docs/TROUBLESHOOTING.md) for common issues

### For Developers

If you're developing or contributing to the Enterprise Support app:

1. **Getting Started:** See [README.md](README.md) for quick start instructions
2. **Contributing:** See [CONTRIBUTING.md](CONTRIBUTING.md) for contribution guidelines
3. **GitHub Issues:** Report bugs or request features via [GitHub Issues](https://github.com/archubbuck/enterprise-support/issues)
4. **GitHub Discussions:** Ask questions or discuss ideas via [GitHub Discussions](https://github.com/archubbuck/enterprise-support/discussions)

## Reporting Issues

### Bug Reports

Found a bug? Please report it via [GitHub Issues](https://github.com/archubbuck/enterprise-support/issues) with:

- Clear description of the issue
- Steps to reproduce
- Expected vs actual behavior
- Device/platform information
- App version
- Screenshots (if applicable)

See our [Bug Report Template](.github/ISSUE_TEMPLATE/bug_report.md) for details.

### Feature Requests

Have an idea for a new feature? Submit it via [GitHub Issues](https://github.com/archubbuck/enterprise-support/issues) with:

- Clear description of the feature
- Use case and benefits
- Any relevant mockups or examples

See our [Feature Request Template](.github/ISSUE_TEMPLATE/feature_request.md) for details.

### Security Issues

**Do not report security vulnerabilities through public GitHub issues.**

Please see our [Security Policy](SECURITY.md) for instructions on reporting security vulnerabilities.
